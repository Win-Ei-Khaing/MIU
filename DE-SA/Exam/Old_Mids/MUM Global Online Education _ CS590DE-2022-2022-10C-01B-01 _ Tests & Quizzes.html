Question 1 of 9 10.0 Points

[10 minutes]

  1. Explain the Brewers CAP theorem.

 

  1. Service Oriented Architecture and a microservice architecture are 2 different architectural styles. Explain clearly in which cases you would apply the Service Oriented Architecture style and in which cases you would apply the microservice architecture style.

a.

Brewers CAP theorem
-Availability
-Consistency
-Partition Tolerance 

When we need availability and consistency(relational databases), we cannot get partition tolerance because data duplication from partitions can make eventual consistency. 

Also when we need availability and partition tolerance(non-relational databases), we cannot get strict consistency because components needs to wait to get the new update data after completed transactions and system is in consistent state.


b.
I would apply Service Oriented Architecture when the system needs few services, not more than 4 or 5 and only simple business logic in ESB because it is hard to manage with SOA with all business logic in ESB.
For the large application which can make ESC complex, I would apply microservice architecture. And it can make easy when we want to use new technologies. And it will avoid monolith's drawbacks.

[10 minutes]

Suppose we need a value object with the name Location which contains the exact location of a vehicle. We express this location with the values longitude and latitude which are integer values.

Write the Java code of the Location value object with the following requirements:

  • The class should be self-validating. The values of longitude, latitude and altitude should always be a positive number. A negative number for longitude, latitude and altitude is not possible.
  • The class has the method addLongitude(int longitudeValue). This method should change the value of longitude. If the given longitudeValue is negative, then the altitude value of the Location object should be decremented with longitudeValue. If the given longitudeValue is positive, then the longitude value of the Location object should be incremented with longitudeValue.
  • The class has the method addLatitude(int latitudeValue). This method should change the value of latitude. If the given latitudeValue is negative, then the altitude value of the Location object should be decremented with latitudeValue. If the given latitudeValue is positive, then the latitude value of the Location object should be incremented with latitudeValue.

Make sure your implementation follows the best practices for a value object we learned in this course.

public class Location{
private int longitude;
private int latitude;

public Location(int longitude, int latitude){
this.longitude=longitude;
this.latitude=latitude;

validate(longitude, latitude);
}

public void validate(int longitude, int latitude){
if(longitude<0 || latitude<0){
throw new InvalidLocationException(); //will be a custom exception
}
}

public addLongitude(int longitudeValue){
return new Location(this.longitude+longitudeValue, this.latitude);
}

public addLatitude(int latitudeValue){
return new Location(this.longitude, this.latitude+latitudeValue);
}
}

Question 7 of 9 10.0 Points

[10 minutes]

Suppose we have 2 microservices, microservice A and microservice B. At a certain point in time microservice A has data that microservice B needs to process. Now we have the following possibilities:

  1. Microservice A calls microservice B using a REST call and sends the necessary data along with the REST call from A to B
  2. Microservice A first calls the registry to get the address of microservice B and then Microservice A calls microservice B using a REST call and sends the necessary data along with the REST call from A to B
  3. Microservice A sends the data to a topic using JMS and microservice B subscribes to that topic.
  4. Microservice A writes the data to a file on the file system. Microservice B polls this location on the file system every 5 seconds, and when a file is written by Microservice A, then microservice B will read the data from the file to process it.

It s expected that in the future, not only microservice B need to process the data from microservice A, but also other microservices will need to process the data from microservice A.

Which of the 4 possibilities would be the best choice. Clearly explain why your choice is the best choice.

The best choice is 3.
Because other microservices will need to process the data from microservice A, we need publish-subscribe flow for that. So, other new microservices subscribe to that topic. Once microservice A processed, those new microservices will automatically know without changing the microservice A.

1 and 2 would need to change microservice A when new other microservices are added. So, they are not good choices.

4 is also a good choice. But if it's compared with 3, it would need to poller for every 5 seconds and need other integration services like file reading which can emerge exceptions like FileNotFound and other more.

So, I think choice 3 is the best choice when we need microservice A's data process not only microservice B and also other microservices.

Question 8 of 9 10.0 Points

[10 minutes]

a. Explain clearly why you should always use a rich domain model and never an anemic domain model

b. Explain clearly the difference(s) between point-to-point and publish-subscribe.

Anemic domain model(class with no business logic)
-No powerful Object Oriented techniques to organic complex business logic
-Hard to find, understand, modify and reuse the business logic/rules because they will spread in other service classes
-software will reflect the data structure of the business, but not the behavioral organization.
-service classes will became too complex with no single responsibility and no separation of concern.

Rich domain model(class with business logic)
-can eliminate all above disadvantages of anemic domain model.

So, we should always use a rich domain model, never an anemic domain model.


Point-to-point
-only one receiver will get the message

Publish-subscribe
-all receivers who subscribed will get the message
